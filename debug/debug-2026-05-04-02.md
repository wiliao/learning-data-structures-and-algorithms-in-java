# Debug Summary — 2026-05-04-02

## Problem

Clicking **"Run"** (or "Run in open file") on `JavaPrimerDemo.java` in VS Code showed a dialog:

> **Build failed, do you want to continue?**

The program could still run if "Continue" was selected (using stale compiled classes), but the incremental build by the Java Language Server (JDT) was failing.

## Root Cause

The root `pom.xml` configured `--enable-preview` in both the compiler and surefire plugins, even though **no Java 25 preview features** are used anywhere in the codebase:

```xml
<!-- pom.xml — problematic configuration -->
<configuration>
    <release>${maven.compiler.release}</release>
    <compilerArgs>
        <arg>--enable-preview</arg>   <!-- ❌ unnecessary -->
    </compilerArgs>
</configuration>
```

When VS Code's Java extension triggers its internal JDT builder (before launching), the Eclipse JDT compiler bundled in `redhat.java` extension does not handle `--enable-preview` properly with Java 25, causing the incremental build to fail.

## Fix

### 1. `pom.xml` — Removed `--enable-preview`

Removed the preview flag from both the **`maven-compiler-plugin`** and the **`maven-surefire-plugin`**:

```diff
- <compilerArgs>
-     <arg>--enable-preview</arg>
- </compilerArgs>

- <configuration>
-     <argLine>--enable-preview</argLine>
- </configuration>
```

### 2. `.vscode/launch.json` (new)

Created a launch configuration so the Java extension uses **Maven** (`mvn compile`) as the pre-launch build step instead of its internal JDT builder:

```json
{
    "version": "0.2.0",
    "configurations": [
        {
            "type": "java",
            "name": "Run JavaPrimerDemo",
            "request": "launch",
            "mainClass": "net.datastructures.chapter01.JavaPrimerDemo",
            "projectName": "chapter01-java-primer",
            "preLaunchTask": "mvn-compile-chapter01"
        }
    ]
}
```

### 3. `.vscode/tasks.json` (auto-generated)

A `mvn-compile-chapter01` shell task was created that runs `mvn compile -f chapter01-java-primer/pom.xml -q`.
